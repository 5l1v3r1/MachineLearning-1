Dataset Information

Base DNS analysis script which tracks and logs DNS queries along with their responses.

Attribute Information

  • ts: Timestamp of request
  • uid: Connection unique id
  • id.orig_h: Originating endpoint’s IP address (AKA ORIG)
  • id.orig_p: Originating endpoint’s TCP/UDP port (or ICMP code)
  • id.resp_h: Responding endpoint’s IP address (AKA RESP)
  • id.resp_p: Responding endpoint’s TCP/UDP port (or ICMP code)
  • proto: Protocol of DNS transaction – TCP or UDP
  • trans_id: 16 bit identifier assigned by DNS client; responses match
  • query: Domain name subject of the query
  • qclass: Value specifying the query class
  • qclass_name: Descriptive name of the query class (e.g. C_INTERNET)
  • qtype: Value specifying the query type
  • qtype_name: Name of the query type (e.g. A, AAAA, PTR)
  • rcode: Response code value in the DNS response
  • rcode_name: Descriptive name of the response code (e.g. NOERROR, NXDOMAIN)
  • AA: Authoritative Answer. T = server is authoritative for query
  • TC: Truncation. T = message was truncated
  • RD: Recursion Desired. T = request recursive lookup of query
  • RA: Recursion Available. T = server supports recursive queries
  • Z: Reserved field, should be zero in all queries & responses
  • answers: List of resource descriptions in answer to the query
  • TTLs: Caching intervals of the answers
  • rejected: Whether the DNS query was rejected by the server
