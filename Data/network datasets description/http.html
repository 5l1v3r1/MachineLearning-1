Dataset Information

Implements base functionality for HTTP analysis. The logging model is to log request/response pairs and all relevant metadata together in a single record.

Attribute Information

  • ts: Timestamp of request
  • uid: Connection unique id
  • id.orig_h: Originating endpoint’s IP address (AKA ORIG)
  • id.orig_p: Originating endpoint’s TCP/UDP port (or ICMP code)
  • id.resp_h: Responding endpoint’s IP address (AKA RESP)
  • id.resp_p: Responding endpoint’s TCP/UDP port (or ICMP code)
  • trans_depth: Pipelined depth into the connection
  • method: HTTP Request verb: GET, POST, HEAD, etc.
  • host: Value of the HOST header
  • uri: URI used in the request
  • referrer: Value of the “referer” header
  • user_agent: Value of the User-Agent header
  • request_ body_len: Actual uncompressed content size of the data transferred from the client
  • response_body_len: Actual uncompressed content size of the data transferred from the server
  • status_code: t Status code returned by the server
  • status_msg: Status message returned by the server
  • info_code: Last seen 1xx info reply code by server
  • info_msg: Last seen 1xx info reply message by server
  • filename: Via the Content-Disposition server header
  • tags: Indicators of various attributes discovered
  • username: If basic-auth is performed for the request
  • password: If basic-auth is performed for the request
  • proxied: Headers that might indicate a proxied request
  • orig_fuids: An ordered vector of file unique IDs from orig
  • orig_mime_types: An ordered vector of mime types from orig
  • resp_fuids: An ordered vector of file unique IDs from resp
  • resp_mime_types: An ordered vector of mime types from resp
